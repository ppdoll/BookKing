# BookKing — 그룹 독서 기록 서비스 요구사항 정리

> 2026-07-13 v2. 원본 요구사항 + 노션 참고 페이지(독서 기록장 PC 버전) 기반.
> 확정된 결정: 위시리스트/독서중 상태 포함, 기록은 그룹별 격리, 그룹 랭킹 + 통합 랭킹,
> 랭킹 최소 3명, soft delete, 초대 링크 7일 만료, 그룹장 위임 기능.
> ❓ 표시는 아직 결정 전 항목 (기본값으로 진행 예정).

---

## 1. 개요

- 그룹 단위 독서 기록 서비스. 그룹원끼리 읽은 책을 등록/공유하고, 그룹 내 책 랭킹을 본다.
- 로그인은 Google OAuth 단일 방식.
- 배포는 Vercel.

## 2. 기술 스택 제안

| 영역 | 제안 | 비고 |
|---|---|---|
| 언어 | **TypeScript** | NestJS는 "언어"가 아니라 Node.js 백엔드 프레임워크. 프론트/백 모두 TypeScript 하나로 통일 가능 |
| 프론트엔드 | **Next.js (React)** | NestJS만으로는 화면을 못 만들기 때문에 프론트 프레임워크가 반드시 필요. Vercel이 Next.js 제작사라 배포 궁합이 가장 좋음 |
| 백엔드 | 옵션 A: **Next.js API Routes / Server Actions** (권장)<br>옵션 B: NestJS를 Vercel 서버리스 함수로 배포 | 옵션 A가 배포·운영이 단순 (프로젝트 1개, 배포 1번). NestJS를 꼭 쓰고 싶으면 옵션 B도 가능하지만 콜드스타트·번들링 설정이 번거롭고 프로젝트가 2개가 됨 |
| DB | **PostgreSQL (Neon 또는 Supabase)** + **Prisma ORM** | Vercel은 서버에 파일/DB를 저장할 수 없으므로 외부 DB 필수. Neon/Supabase 둘 다 무료 티어 있음 |
| 인증 | **Auth.js (NextAuth)** + Google Provider | 세션 관리까지 포함. Supabase를 쓰면 Supabase Auth로 대체 가능 |
| 책 검색 | **네이버 검색 API (책)** — `https://openapi.naver.com/v1/search/book.json` | 아래 6장 참고. 쇼핑 검색 페이지 크롤링은 비권장 |
| 이미지 | 네이버 API가 주는 표지 썸네일 URL을 그대로 사용 | 별도 이미지 스토리지 불필요 (직접 업로드 허용 시 Vercel Blob 필요 ❓) |

**결론: 추가로 필요한 것 = React/Next.js(프론트), PostgreSQL(DB), TypeScript(공통 언어).**

## 3. 역할(권한)

역할은 **그룹별**로 부여된다 (한 사람이 A그룹 그룹장 + B그룹 사용자일 수 있음).

| 역할 | 권한 |
|---|---|
| 그룹장 | 모든 기능. 운영자 지정/해제. 그룹 관리. **그룹장 위임** (다른 그룹원에게 OWNER 이전) |
| 운영자 | 사용자 기능 + 그룹 내 글(독서 기록) 검색·수정·삭제 |
| 사용자 | 자기 기록 등록/수정/삭제, 책장, 검색, 랭킹 조회, 새 그룹 생성 |

## 4. 가입 / 로그인 플로우

1. 그룹장이 되려는 사람은 회원가입(Google 로그인) 후 그룹 생성.
2. 그룹 생성 시 그룹 고유 초대 링크 발급 (예: `/join/{inviteCode}`).
   - 초대 링크는 **발급 후 7일 만료**. 만료되면 그룹장이 새 링크 재발급 (기존 링크는 무효화).
3. 초대 링크로 접속 → Google 로그인 → 해당 그룹에 "사용자" 역할로 가입.
4. 최초 가입 시 내부 고유 UUID 발급 (Google 계정과 매핑, 화면에 표시).
5. 최초 가입 시 사용자 이름 등록 필수 (미등록 시 다음 단계 진행 불가).

## 5. 메뉴 / 페이지

### 공통 레이아웃
- **TOP**: 사이트 정보(로고), 그룹 선택 SELECT BOX, 사용자 정보(이름 + UUID)
- **MAIN**: 노션 독서 기록장 스타일
  - 노션 참고 페이지 구성: 읽을 예정(위시리스트) → 독서중 → 독서 완료 상태 플로우, 나의 독서 리스트, 올해의 독서 기록 차트, 전체 보기 테이블(별점별/장르별 보기)
- **우측 사이드바**: 랭킹 (그룹 랭킹 / 통합 랭킹 탭 전환)
  - **책 랭킹**
    - 그룹 랭킹: 현재 선택된 그룹의 완독 기록만 집계
    - 통합 랭킹: 서비스 전체 그룹의 완독 기록 집계 (책 제목·표지·평균 별점·읽은 수만 노출, 개별 기록/작성자는 비공개 — 그룹 격리 원칙 유지)
    - 정렬 기준: 별점 순 (평균 별점) / 많이 읽어진 순 (완독한 사람 수)
    - **최소 3명 이상**이 완독한 책만 랭킹에 포함
  - **유저 랭킹 (독서왕)**: 완독 권수 많은 순 유저 랭킹
    - 그룹 탭에서만 표시 (통합 탭에서 유저를 노출하면 그룹 격리 원칙과 충돌하므로 제외 — 필요 시 익명화해서 추후 검토)
    - 기간 기준: 올해 완독 수 (연간 리셋) ❓ 전체 누적으로 할지 결정 필요

### 독서 상태 플로우 (노션 참고 페이지와 동일)

```
읽을 예정 (WISH) → [독서 시작] → 독서중 (READING) → [독서 완료] → 완독 (DONE)
```

- 책 등록 시 기본 상태는 "읽을 예정" (바로 완독으로 등록하는 것도 가능)
- [독서 시작] 버튼 → 당일 날짜가 시작일에 자동 입력, 독서중으로 이동
- [독서 완료] 버튼 → 당일 날짜가 완료일에 자동 입력, 완독으로 이동
- 별점·기억에 남는 문장·느낀 점은 완독 시점에 입력 (이후 수정 가능)
- 랭킹 집계는 **완독(DONE) 기록만** 대상

### 기록 공개 범위

- 읽은 책 기록은 **등록한 그룹에만 보임** (A그룹 기록은 B그룹에서 안 보임)
- 통합 랭킹은 책 단위 집계 수치만 노출 (위 참고)

### 사용자 메뉴
1. 그룹 선택 (가입한 그룹 목록 중 선택, 선택된 그룹 기준으로 화면 구성)
2. 책 등록 (7장 필드 참고 — 읽을 예정/독서중/완독 상태 선택 가능)
3. 내가 읽은 책 책장
   - 책 제목 + 표지 썸네일 그리드, 상태별(읽을 예정/독서중/완독) 구분
   - 정렬: 읽은 날짜 / 평가(별점) / 제목 / 저자
   - 내 기록 선택 → 수정/삭제/상태 변경
4. 책 검색
   - 검색 대상: ① 현재 그룹에 등록된 기록 + ② 네이버 책 검색 API 결과
5. 나만의 그룹 생성

### 운영자 메뉴 (사용자 +)
- 글 관리 페이지: 사용자 이름 / 책 제목으로 그룹 내 기록 검색 → 삭제 (수정도 가능)
- 삭제는 **soft delete** (`deletedAt` 기록) — 목록에서 숨겨지고, 그룹장이 복구 가능

### 그룹장 메뉴 (운영자 +)
- 운영자 관리 페이지: 그룹원 목록에서 운영자 지정/해제
- 초대 링크 재발급 (7일 만료)
- 그룹장 위임: 그룹원 중 한 명에게 OWNER 이전 (본인은 운영자로 전환)
- 삭제된 글 복구

## 6. 네이버 책 검색 연동

요구사항에 적힌 URL(`search.shopping.naver.com/book/search...`)은 **웹페이지**라서 크롤링해야 하는데, 구조가 자주 바뀌고 차단 위험이 있으며 약관 위반 소지가 있음. 대신 **네이버 공식 오픈 API**를 권장:

- 엔드포인트: `GET https://openapi.naver.com/v1/search/book.json?query=...`
- 발급: [네이버 개발자센터](https://developers.naver.com)에서 애플리케이션 등록 → Client ID / Secret (무료, 일 25,000회)
- 응답에 포함: **제목, 저자, 출판사, 표지 이미지(썸네일), 링크, 가격(discount), ISBN, 출간일** — "책정보 생성" 버튼에 필요한 값 전부 제공
- ISBN을 받을 수 있어서 "같은 책" 판별(랭킹 집계)에 그대로 사용 가능

**"책정보 생성" 버튼 동작**: 입력된 제목(+저자, 출판사)으로 API 검색 → 후보 목록 표시 → 사용자가 선택 → 썸네일/링크/가격/ISBN 자동 입력.

## 8. 데이터 모델 초안

```
User          : id(UUID), googleId, email, name, createdAt
Group         : id, name, inviteCode, inviteExpiresAt(발급+7일), ownerId, createdAt
GroupMember   : userId, groupId, role(OWNER/ADMIN/MEMBER), joinedAt
Book          : id, isbn(unique), title, author, publisher, thumbnailUrl,
                link, price   ← 네이버 API 결과를 공유 엔티티로 저장
ReadingRecord : id, userId, groupId, bookId,
                status(WISH/READING/DONE),
                startDate, endDate, rating(0~5, 0.5),
                recommendMbti, memorableQuote, review,
                deletedAt(soft delete), deletedBy,
                createdAt, updatedAt
```

- 그룹 랭킹: 해당 `groupId`의 DONE 기록을 `bookId`로 집계, 완독자 3명 이상만.
- 통합 랭킹: 전체 DONE 기록을 `bookId`로 집계, 완독자 3명 이상만 (집계 수치만 노출).
- soft delete된 기록은 모든 목록/랭킹에서 제외, 그룹장이 복구 가능.
- ISBN이 없는 책(직접 입력)은 제목+저자로 매칭하거나 별도 Book 행 생성.

## 8.5 그룹 생성 이용권(슬롯) 시스템 (2026-07-14 추가)

- **기본 1개 무료**: 모든 유저는 그룹을 1개까지 무료로 생성. 초대 링크 가입은 무제한.
- **이용권 차감 기준**: "생성한 그룹 수" (`Group.createdById`, 불변) — 그룹장을 위임해도 이용권은 반환되지 않음 (악용 방지).
- **쿠폰**: 코드 입력 시 이용권 지급. 슬롯 수·사용 횟수·유효기간 설정 가능, 1인 1회.
- **이용권 요청 흐름**: 사용자가 🎟️ 이용권 페이지에서 개수·사유와 함께 요청 (대기 중 요청은 1건만)
  → 사이트 관리자가 🛠 사이트 관리에서 **지급 슬롯 수를 정해 쿠폰 발급·승인** 또는 거절
  → 요청자는 이용권 페이지에서 결과·쿠폰 확인 후 [바로 사용하기].
- **사이트 관리자**: `User.isSiteAdmin` 플래그 또는 `ADMIN_EMAILS` 환경변수. 현재 ppdolla@gmail.com 지정됨.
- **유저 관리·제재 (2026-07-20 추가)**: 사이트 관리에서 전체 가입 유저 목록(이름·이메일·가입일·그룹/기록 수, 이름·이메일 검색, 20개+더 보기) 확인.
  **계정 정지**(`User.suspendedAt/suspendedReason`) — 정지된 유저는 로그인해도 모든 페이지·액션이 `/suspended` 안내(사유 표시)로 차단, 해제 시 즉시 복구.
  본인·다른 사이트 관리자는 정지 불가.
- **현금 결제(수동)**: 송금 확인 후 사이트 관리에서 쿠폰을 만들어 코드 전달. PG(카드결제) 연동은 사업자등록 후 2단계로.
- 데이터 모델: `SlotGrant`(지급 내역, 1행=1슬롯), `Coupon`, `CouponRedemption`(1인 1회), `SlotRequest`(요청·처리 상태).

## 8.55 개인 책장 (2026-07-14 추가)

- **가입(이름 등록) 완료 시 『○○의 책장』 1인 그룹 자동 생성** — 신규 유저가 그룹 없이도 바로 기록 가능
- `Group.isPersonal = true`: **이용권 미차감**(기본 무료 그룹 1개 정책과 별개), 초대 링크·검색 노출·가입 승인 불가(1인 전용), 그룹 관리·가입 신청 메뉴 숨김
- 개인 책장의 기록도 통합 랭킹 집계에는 포함 (익명 수치)
- 기존 유저 전원 백필 완료 (2026-07-14)

## 8.6 그룹 옵션 & 그룹 검색 (2026-07-14 추가)

- **그룹 옵션** (그룹장이 그룹 관리에서 설정, 기본 모두 OFF):
  - **🔍 외부 검색 허용** (`searchable`): 그룹 찾기에 노출. 노출 정보는 그룹 이름·그룹장 이름·멤버 수·기록 수뿐 (기록 내용은 가입 후에만 — 격리 원칙 유지)
  - **👀 보기 전용** (`readOnly`): 일반 그룹원은 열람만, 기록 등록·수정·상태 변경은 **그룹장·운영자만**. 추천 도서 공지용
  - **🙋 가입 승인제** (`joinApproval`): 초대 링크·검색 어느 경로든 가입이 **신청 → 그룹장·운영자 승인** 흐름으로 바뀜. 승인 처리 화면은 `/admin/joins`(운영자 이상, 상단 🙋 가입 신청 + 대기 배지). 승인 시 자동 가입, 거절 이력 유지 (`GroupJoinRequest`, 그룹×유저 unique)
- **그룹 찾기** (`/groups/search`): 공개 그룹 이름 검색 + 최신순 목록, [가입하기] 즉시 가입
- **미가입 유저 기본 화면**: 가입한 그룹이 없으면 홈 대신 그룹 찾기(환영 배너)로 이동
- **멤버 관리 보강** (공개 가입에 따른 안전장치):
  - 그룹장 **내보내기(강퇴)** — 그룹 관리 멤버 목록에서. 내보내도 그 사람의 기록은 그룹에 남음
  - 본인 **그룹 나가기** — 홈 하단. 그룹장은 위임 후에만 나갈 수 있음

## 8.7 서점 제휴 링크 (2026-07-21 추가)

- 기록 상세·책 검색에 **쿠팡·알라딘·예스24·교보문고 구매 링크** 표시
